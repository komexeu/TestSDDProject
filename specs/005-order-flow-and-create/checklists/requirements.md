# 005 合併規格：點餐流程與建立訂單 - 需求檢查清單

## 基本需求
- [ ] 用戶可於Line介面選擇餐點並送出訂單
- [ ] 櫃檯可接收並確認訂單
- [ ] 廚房可接收訂單並通知櫃檯
- [ ] 用戶訂單狀態依流程自動更新
- [ ] 櫃檯可標記餐點完成，通知用戶取餐
- [ ] 用戶完成取餐後，狀態顯示「已取餐完成」
- [ ] 訂單資料驗證（收件資訊、商品數量、金額等）
- [ ] 商品庫存檢查與同步
- [ ] 訂單建立成功後發送通知
- [ ] 異常/例外情境處理（取消、重複送單、庫存不足、付款失敗等）
- [ ] 訂單資料永久保存（除非法規另有規定）
